Summary

In this project, we’ll be utilizing the Navier-Stokes equations to simulate an incompressible and homogeneous fluid. In particular, we’ll be attempting to simulate smoke with features including adjustable variables and the ability to add objects into the simulation.

Problem Description

From cigarette butts to a campfire, the need to be able to render and simulate smoke is extremely prominent. We’ll be providing a sandbox environment that simulates smoke. The smoke will be outputted from either the mouse cursor or given points set by the user. The sandbox will also contain many adjustable variables including pressure, temperature, and vorticity, that we’ll be able to adjust and see the resulting effect on the smoke. On top of that, our sandbox will have a feature that allows for the placement of objects inside the frame. We’ll then simulate how the smoke bounces or wraps around these objects. The simulation will revolve around the Navier-Stokes equations and will be utilizing the frameworks three.js and WebGL.

Goals and Deliverables

What we plan to deliver

In order to have a successful project, we plan on delivering a demo that can generate an accurate simulation of smoke within a controlled system. This demo will allow the user to input smoke into the system and see how it interacts with itself and eventually fades. We also plan to deliver an analysis of the concessions that must be made in order to make the demo run effectively. That is, our analysis will concern the balance between an ideally realistic simulation and one that is computationally simple enough to produce within the context of a demo that is actively running instead of being pre-computed, as would be the case with a render. We plan on accomplishing this by using the Navier-Stokes equations to create an interactive web application primarily powered by JavaScript frameworks and WebGL.

What we hope to deliver

The implementation by Jos Stam of smoke shaders that we plan to follow prioritizes computational speed over physical accuracy, which is ideal for demos and games but may miss some aspects of smoke’s real-world behavior. We hope to implement a modified algorithm that better models real-world behavior in order to compare the two and potentially suggest improvements to Stam’s implementation that account for these inaccuracies. We also hope to implement the ability to add physical objects and forces into the scene. For instance, we would like the user to be able to add fans or solid objects with which the smoke can interact and have differing effects given the nature of the object.
